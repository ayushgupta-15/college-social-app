# Architecture

CampusConnect relies on a decoupled, scalable microservice-like structure running as a monolith on Render, with a dedicated mobile application consuming its APIs and WebSockets.

## High-Level Flow

```text
┌──────────────────────────────┐
│     React Native (Expo)      │  ← TypeScript, Axios, React Navigation v7
└──────────────┬───────────────┘
               │ HTTPS / WebSocket
┌──────────────▼───────────────┐
│      Go + Gin REST API       │  ← Firebase Admin token verification middleware
│  Auth │ Feed │ Groups│ Events │
│  Chat │ Users│ Notifs│       │
└──┬──────────────────────┬────┘
   │                      │
┌──▼────────┐      ┌──────▼──────┐
│ PostgreSQL│      │    Redis    │
│ (12 migs) │      │ (session/   │
└───────────┘      │  feed cache)│
                   └─────────────┘
               Firebase Auth / FCM
```

## Tech Choices

### React Native & Expo
React Native allows for rapid cross-platform development across iOS and Android from a single codebase. Expo accelerates the process with simplified build workflows and out-of-the-box libraries for secure storage and navigation.

### Go & Gin
Go provides a high-concurrency, low-overhead backend that is extremely type-safe. The Gin framework was chosen for its minimalistic footprint and high routing performance.

### PostgreSQL & Redis
PostgreSQL is the source of truth for the highly relational data generated by a social network (followers, groups, event registrations, likes). Redis is used to reduce database load for session management and cache high-read endpoints like the global feed.
